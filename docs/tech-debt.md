# Technical Debt Register

This document tracks compromises, shortcuts, or design issues in the OpenBench codebase that require remediation.

| ID | Category | Title | Description / Context | Impact | Remediation Plan | Effort | Status |
|----|----------|-------|-----------------------|--------|------------------|--------|--------|
| TD-001 | Architecture | Implicit Env File Discovery | `apps/backend/internal/config/config.go` still selects `.env` files from the current directory, `..`, or `../..`. This is narrower than the original parent-tree walk, but startup behavior still depends on the process working directory. | Local, CI, and container runs can load different env files unless launched from expected directories. | Resolve env files from an explicit path or fixed project-root convention. | Medium | Active |
| TD-002 | Performance | Database Pool Observability | `apps/backend/internal/database/database.go` exposes `Stats()`. **Partially mitigated:** `main.go` now logs `db.Stats()` every 30s. No metrics endpoint or alerting path yet. | Latency regressions still hard to distinguish between pool starvation and slow queries. | Emit `db.Stats()` through dedicated metrics endpoint and add slow query instrumentation. | Medium | Active |
| TD-003 | Database | Missing Refresh Token Cleanup | No background worker cleans up expired or revoked refresh tokens in database. | `refresh_tokens` table grows indefinitely, degrading query performance. | Implement periodic goroutine or DB cron to purge expired/revoked refresh tokens. | Low | Active |
| TD-004 | Configuration | Environment Name Mismatch in Exemption Check | `config.go` skips validation only for `"development"`/`"test"`, but `main.go` also recognizes `"dev"`. `APP_ENV=dev` triggers prod validation incorrectly. **Resolved:** `config.go` now remaps `"dev"` → `"development"` before validation. | History item — no longer an active issue. | Completed: `config.go` lines 53-54 normalize `"dev"` to `"development"`. | Low | Resolved |
| TD-005 | Security | Refresh Error Detail Leakage | `auth/handler.go` returns specific error types (compromised/revoked/expired) to client, aiding token state enumeration. | Low risk (all 401), but aids enumeration. | Consolidate to generic "invalid refresh token" response. | Low | Active |
| TD-006 | Build | Stale go.sum Entries | `go.sum` contains entries for `lib/pq` and `go-sqlite3` not in `go.mod`. Leftovers from dependency changes. | None functional, but `go.sum` larger than needed. | Run `go mod tidy`. | Trivial | Active |
