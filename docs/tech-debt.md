# Technical Debt Register

This document tracks compromises, shortcuts, or design issues in the OpenBench codebase that require remediation.

| ID | Category | Title | Description / Context | Impact | Remediation Plan | Effort | Status |
|----|----------|-------|-----------------------|--------|------------------|--------|--------|
| TD-001 | Architecture | Implicit Env File Discovery | `apps/backend/internal/config/config.go` still selects `.env` files from the current directory, `..`, or `../..`. This is narrower than the original parent-tree walk, but startup behavior still depends on the process working directory. | Local, CI, and container runs can load different env files unless launched from expected directories. | Resolve env files from an explicit path or fixed project-root convention. | Medium | Active |
| TD-006 | Build | Stale go.sum Entries | `go.sum` contains entries for `lib/pq` and `go-sqlite3` not in `go.mod`. Leftovers from dependency changes. | None functional, but `go.sum` larger than needed. | Run `go mod tidy`. | Trivial | Active |
| TD-008 | Testing | Direct Integration Test Ryuk Override | Rootless Podman environments may still need `TESTCONTAINERS_RYUK_DISABLED=true` when running integration tests directly outside the Makefile path. | IDE or direct `go test -tags=integration ./...` runs can fail on runtimes where Ryuk cannot start, even though the Makefile supports explicit override. | Document the direct-test override or add runtime-aware helper docs for non-Make invocations. | Low | Active |
| TD-010 | API Contract | Frontend Ticket Payload Drift | Ticket create/update forms send fields outside `TicketCreate`/`TicketUpdate` schemas, relying on backend ignoring unknown JSON fields. | Future strict validation or generated clients can break create/update flows. | Type service requests from generated OpenAPI request schemas and trim payloads to documented fields only. | Medium | Active |
| TD-014 | API Contract | Empty List Response Omits `data` | `apps/backend/internal/pkg/response/response.go` still tags `APIResponse.Data` with `omitempty`. The admin tickets UI now defensively normalizes missing data, but the backend can still serialize successful empty list responses without the `data` key. | API consumers can receive a successful list response with no `data`, causing contract drift and making empty-state regressions easier to reintroduce. | Remove `omitempty` from successful response data or introduce a list response helper that always serializes `data: []`; add backend tests for empty list serialization and frontend service tests for missing/null data compatibility. | Low | Active |
| TD-015 | Performance | POS Sales Archive N+1 Item Loading | `apps/backend/internal/sales/repository.go` lists sales and then fetches `sale_items` once per sale, while migration `000007` does not add an index on `sale_items.sale_id`. | Sales archive reads will degrade as transactions grow, and each list page can trigger repeated table scans. | Add an index on `sale_items.sale_id` and consider replacing per-sale item lookups with one joined/batched query. | Low | Active |
| TD-016 | Testing | POS Mock Receipt Prefix Drift | Mock sales still seed/generate `OB-INV-...` invoice numbers while the backend generates `INV-YYYYMM-####`; the E2E assertion only checks `INV-`, which matches both formats. | Mock-mode browser tests can pass while user-visible receipt/archive output differs from the real backend. | Align mock invoice generation and seed data with backend format, then assert an anchored invoice pattern in E2E. | Low | Active |
